Cuestionario sobre Ramas en Git

  1. ¿Qué es una rama (branch) en Git y para qué sirve?

    Es una línea de tiempo separada del proyecto principal. Sirve para desarrollar nuevas funciones o realizar prácticas sin afectar el código estable en la rama main.

  2. ¿Cómo se crea y se cambia entre ramas en Git?

    Se crea con el comando git branch nombre_rama y se cambia a ella con git checkout nombre_rama.

  3. ¿Cuál es la diferencia entre la rama main y otras ramas?

    La rama main es la rama principal y definitiva del proyecto. Las demás son ramas auxiliares para trabajo temporal o experimental.

  4. ¿Cómo se combinan ramas en Git (merge)?

    Se debe estar en la rama que recibirá los cambios y usar el comando git merge nombre_de_la_rama_a_unir

  5. ¿Qué problemas pueden ocurrir al trabajar con ramas (conflictos) y cómo se resuelven?

    Ocurren cuando dos ramas modifican la misma línea de un archivo. Se resuelven editando manualmente el archivo en conflicto para decidir qué cambios conservar y luego haciendo un nuevo commit.
